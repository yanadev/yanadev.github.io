ES6 变量解构赋值

发表于 2020-03-31 更新于 2022-05-23 分类于 ES6 ， JS扩展 阅读次数：
本文字数： 4.6k

解构( Destructuring )

从数组和对象中提取值，对变量进行赋值

before : 变量赋值只能直接指定值

ES6 : 可从数组中提取值，按照对应的位置，对变量赋值

  • 该写法本质上属于 模式匹配：只要等号两边模式相同，左边的变量就会被赋予对应的值。

数组的解构赋值

嵌套数组进行解构

①：解构不成功，变量的值为undefined

②：不完全解构但成功。即等号左边只匹配一部分等号右边的数组，此时，解构依旧成功

  • 只要某种数据结构具有Iterator（迭代器）接口，就可以采用数组形式的解构赋值

默认值

解构赋值允许指定默认值（等号左边）

ES6 内部使用严格相等运算符（===），判断一个位置是否有值
所以，只有当一个数组成员严格等于undefined，默认值才会生效

如果默认值是一个表达式，那么表达式惰性求值，即只有在用到的时候才会求值

因为x可以取到值，所以函数 f 根本不会执行

默认值可以引用解构赋值 的其他变量，但该变量必须已经声明

对象的解构赋值

与数组的区别

数组的元素是按次序排列的，变量的取值由它的位置决定；
而对象的属性是没有次序的，变量必须与属性同名，才能取得正确的值。
如果解构失败，变量的值等于 undefined

优点

可以很方便地将现有对象的方法，复制到某个变量上

注意

如果变量名与属性名不一致们必须写成一下形式

内部机制

先找到同名属性，然后再赋给对应的变量；
真正被赋值地是后者，而不是前者

  • foo 是匹配模式，baz 才是变量

用于嵌套结构的对象

分别是对loc、start、line三个属性的解构赋值。注意，最后一次对line属性的解构赋值之中，只有line是变量，loc和start都是模式，不是变量。

嵌套赋值

子对象所在的父属性不存在，那么将会报错。

对象的解构赋值也可以指定默认值

  • 默认值生效的条件是：对象的属性值严格等于 undefined.

注意点

  1. 如果要将一个已经声明的变量用于解构赋值，必须非常小心

语法错误，因为 JavaScirpt 引擎会将 {x} 理解成一个代码块
只有不把大括号写在行首，避免 Javascirpt 将其解释成代码块，才能解决这个问题

  1. 解构赋值允许等号左边的模式中，不放置任何变量名=====>可以写出非常古怪的赋值表达式
  1. 可以对数组进行对象属性的解构====>数组的本质是特殊的对象

字符串的解构赋值

字符串被转换成一个类似数组的对象=====>可以解构赋值

类似数组的对象都有一个 length 属性====>可以对该属性解构赋值

数值和布尔值的解构赋值

等号右边是数值或者布尔值的时候，会先转成对象

函数参数的解构赋值

函数参数也可以使用解构赋值

函数 add 参数表面上是一个数组，但在传入参数的那一刻，数组参数就被结构成变量 x 和 y 。对于函数内部的代码而言，它们能感受到的参数就是 x 和 y 。
