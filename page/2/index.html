本文字数： 21k

将构建配置抽离成 npm 包的意义

  • 通用性
    • 业务开发这无需关注构建配置
    • 统一团队构建脚本
  • 可维护性
    • 构建配置合理的拆分（开发、生产、ssr环境）
    • README文档 changelog 文档等
  • 质量
    • 冒烟测试、单元测试、测试覆盖等
    • 持续集成

可选方案

  • 通过多个配置文件管理不同环境的配置
    • 通过webpack –config参数来控制
  • 将构建配置设计成一个库
  • 抽成一个工具进行管理
  • 将所有配置都放在一个文件，通过 –env 参数控制分支选择
