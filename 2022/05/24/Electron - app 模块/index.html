Electron - app模块

发表于 2022-05-24 分类于 Electron ， app模块 阅读次数：
本文字数： 2.6k

app

事件列表

  • will-finish-launching 应用完成基础启动时触发

    • 不同操作系统有差别，window 和 linux 下该事件与 ready 相同
    • OS X 中，相当于 NSApplication中的 applicationWiilFinishLaunching 提示
    • 应该经常在这里为 open-file 和 open-url 设置监听器，并启动奔溃报告和自动更新

  • ready 完成初始化时触发

  • window-all-closed 所有窗口均被关闭时触发

  • before-quit 应用程序开始关闭它的窗口时被触发

    • 返回 event Event，调用 event.preventDefautl() 将阻止应用程序的默认行为

  • will-quit 应用程序正在退出时触发

    • 返回 event Event, exitCode Integer

  • ‘open-file’ OS X 用户想在应用中打开一个文件时触发

    • 返回 event Event,path String

  • ‘open-url’ OS X 用户想在应用中打开一个链接时触发

    • 返回 event Event,path String

  • ‘active’ OS X 应用被激活时触发，点击应用的 dock 图标时

    • 返回 event Event, hasVisibleWindows Boolean

  • browser-window-blur 当一个 BrowserWindow 失去焦点时触发

    • 返回 event Event, window BrowserWindow

  • browser-window-focus 当一个 BrowserWindow 获得焦点时触发

    • 返回 event Event, window BrowserWindow

  • browser-window-created 当一个 BrowserWindow 被创建时触发

    • 返回 event Event, window BrowserWindow

  • certificate-error 对 url 校验 certificate 证书失败时触发，若需要信任该证书，需要阻止默认行为并调用回调 （event.preventDefault() 之后 执行 callback(true)）

    • event Event
    • WebContents WebCOntents
    • url String - URL 地址
    • error String - 错误码
    • certificate Object
    • data Buffer - PEM 编码数据
    • issueName String - 发行者的共有名称
    • callback Function

  • select-client-certificate 当客户端认证被请求时触发,需要阻止默认事件防止应用自动使用第一个证书进行校验

    • event Event
    • WebContents WebCOntents
    • url String - URL 地址
    • error String - 错误码
    • certificateList[Object]
    • data Buffer - PEM 编码数据
    • issueName String - 发行者的共有名称
    • callback Function

  • login 当 WebContents 要进行一次 HTTP 登录验证时触发

  • gpu-process-crashed 当 GPU 进程奔溃时触发

方法列表

有些方法只能用于特定的操作系统

  • app.quit() 视图关掉所有窗口

    • before-quit
    • will-quit

  • app.hide() 隐藏所有应用窗口，不是最小化

  • app.show() 隐藏后重新显示所有窗口，不会自动选中

  • app.exit(exitCode) 携带 exitCode 退出应用，立刻关闭，不会询问用户

    • exitCode Integer
    • 不会触发 before-quit 和 will-quit 两个事件

  • app.getAppPath() 获取当前应用所在的文件路径

  • app.getPath(name) 返回与 name 参数相关的特殊文件夹或者文件路径，失败时抛出一个 error

    • name String
      • home 用户的home 文件夹（主目录）
      • appData 当前用户的应用数据文件夹，默认对应
        • %APPDATA% —– windows
        • $XDG_CONGIF_HOME or ~/.config —— Linux
        • ~/Library/Application ——— Support OS X 中
      • userData 储存应用程序设置文件的文件夹，默认是 appData 文件附加应用的名称
      • temp 临时文件夹
      • exe 当前可执行文件
      • module libchromiumcontent 库
      • desktop 当前用户的桌面文件夹
      • documents 用户文档目录的路径
      • downloads 用户下载目录的路径
      • music 用户音乐目录的路径
      • pictures 用户图片目录的路径
      • videos 用户视频目录的路径

  • app.setPath(name, path) 重写某个 name 路径为 path

    • name String
    • path String

  • app.getVersion() 返回加载应用程序的版本，若 package.json 中没有指明 version，则返回当前包或可执行文件的版本

  • app.getName() 返回当前应用程序的 package.json 中的名称

  • app.setName(name) 重写当前应用的名字

    • name Stirng

  • app.getLocal() 返回当前应用程序的语言

  • app.addRecentDocument(path) OS X windows 在最近访问的文档列表中添加 path

    • path String

  • app.clearRecentDocuments() OS X windows 清除最近访问的文档列表

  • app.setUserTasks(tasks) Windows

    • tasks[Task] 一个由 Task 对象构成的数组
    • 将 tasks 添加到 windows 中 JumpList 功能放入 Tasks 分类中
    • Task Object
      • program String 执行程序的路径
      • arguments String 当 program 执行时的命令行参数
      • title String JumpList 中显示的标题
      • description String 对这个任务的描述
      • inconPath String JumpList 中显示的图标的绝对路径，可以是一个任意包含一个图标的资源文件，可通过指明 process.execPath 来显示程序中的图标
      • iconIndex Integer 图标文件中的采用的图标位置
        • 若一个图标文件中包括了多个图标，需要设置该值，以确定使用哪一个图标
        • 若图标文件中只包含一个图标，该值为 0
