webpack5-资源内联

发表于 2022-03-31 更新于 2022-08-19 分类于 webpack ， inline source ， 资源内联 阅读次数：
本文字数： 6.2k

资源内联（inline source）

将一个资源以内联的方式嵌入另一个资源里面

*HTML 内联 CSS：即内联 CSS / 行内 CSS *

  • 写几行 reset css 通过 style 标签嵌入 html
  • css 内联图片：将小图片通过 base64 编码方式内嵌到 css 中
  • 可以搜索小 icon 内联到 css 中

意义

  • 工程维护
  • 页面加载性能
  • 页面加载体验

工程维护

多个入口，信息重复，一处维护

  • 多页打包项目会借助 html-webpack-plugin，每个页面都有一个对应的 html 模板
  • 每个模板有很多相似的内容：meta信息、SSR时需要的一些占位符

推荐方式是 维护一份 meta.html ，将该内容内联到每个模板中去

图片、字体内联，减少请求次数

很多时候，小图的内联都是直接使用 base64 引入到 css 中，会导致代码的可读性下降

页面加载性能

减少 HTTP 请求（若网站使用 http2，这点的意义可能不是很大）

将各种小图、小字体（<5k），在生产环境 base64 到代码里可极大减少页面的请求数量，从而提高页面的加载时间

页面加载体验

浏览器解析 HTML 源码，是 从上到下 解析

因此会把

  • css 放头部
  • JS 放底部

SSR（服务端渲染）场景为例，如果不将打包出来的 css 内联到 HTML 中，HTML 出来的时候，页面结构已经有了，还需要去发送一次请求去请求 css，会出现页面闪烁、网路差的时候更加明显

内联类型

  • HTML 内联
  • CSS 内联
  • JS 内联
  • 图片、字体内联

HTML 、JS 内联

  • 安装 raw-loader
  • 安装 lib-flexible.js 库

  • 在使用了 html-webpack-plugin 生成 html 模板的情况下，默认使用的 ejs 模板引擎 ，支持 <%= %>

  • 修改 webpack.prod.js

  • /public 目录下创建 meta.html 静态文件
  • 将资源内联进 index.ejs(插件生成的模板) 内联 js 和 html

CSS 内联

为了更好的加载体验，将打包好的 CSS 内联到 HTML 头部，这样 HTML 记载完成 CSS 就可以直接渲染出来，就可以避免页面闪动的情况了

核心思路

将页面打包过程产生的所有 CSS 提取成一个独立文件，将这个 CSS 文件内联进 HTML head 里面

  • mini-css-extract-plugin
  • html-inline-css-webpack-plugin

html-inline-css-webpack-plugin 需要放在 html-webpack-plugin 之后，需要先有模板才能将 css 在嵌入html模板

图片、字体内联

  • 可借助 url-loader，配置小于某个大小的图片/字体文件，在构建阶段转成 base64
